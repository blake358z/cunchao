# 村超运营管理后台与后端能力 PRD

版本：v0.1｜日期：2026-06-16｜阶段：管理后台 MVP

## 1. 建设目标

管理后台面向内容运营、赛事数据员、文旅运营、社区审核员、球队管理员和系统管理员，支撑村超 H5、小程序和未来 App 的日常运营。首版目标是把资讯上新、赛事管理、球队球员资料、文旅攻略、社区审核、数据看板和系统设置组织成一个可用后台，并预留后续数据库、权限和外部数据源接入。

## 2. 后台模块

| 模块 | 使用角色 | 首版能力 | 后续生产化 |
|---|---|---|---|
| 总览 | 项目负责人、运营负责人 | DAU、观看时长、资讯 CTR、预约转化、内容池规模、模块健康度 | 接入真实埋点仓库和运营告警 |
| 内容上新 | 内容运营 | 创建资讯/视频/战报/攻略/公告草稿，查看审核队列 | CMS 草稿、富文本、定时发布、版本回滚 |
| 赛事管理 | 赛事数据员 | 查看未来比赛、赛程核验、模块化赛事数据结构 | 赛程导入、比分事件录入、积分榜自动计算 |
| 球队球员 | 赛事运营、球队管理员 | 球队资料库、球队/球员信息字段规划 | 球队认领、资料纠错、球员授权信息 |
| 文旅攻略 | 文旅运营 | 按未来比赛匹配村寨攻略、住宿餐饮、图片素材 | 商户后台、线路编辑、攻略复用和图片版权管理 |
| 社区审核 | 社区运营 | 帖子/评论巡检、治理能力清单 | 敏感词、举报工单、禁言、删除、置顶、精选 |
| 数据看板 | 产品、运营、增长 | 页面 PV、平均时长、CTR、转化漏斗、事件字典 | 接入 PostHog/自建 ClickHouse/数据仓库 |
| 系统设置 | 系统管理员 | 角色权限、数据源、发布流、审计日志规划 | RBAC、API 密钥、审计、灰度发布 |

## 4. 后端接口

### 管理接口

| 接口 | 方法 | 用途 | 当前状态 |
|---|---|---|---|
| `/api/admin/overview` | GET | 管理后台总览、模块健康度、漏斗和事件字典 | 已开发，当前由 mock/现有数据推导 |
| `/api/admin/content/drafts` | POST | 创建内容草稿并进入审核队列 | Fastify 已开发，Vercel 静态版暂为前端本地草稿 |
| `/api/analytics/events` | POST | 接收页面浏览、点击、预约、评论、收藏等埋点 | 已开发，当前无状态接收 |

### 埋点事件字典

| 事件 | 触发场景 | 关键属性 | 指标用途 |
|---|---|---|---|
| `page_view` | 进入首页、赛事、详情、社区、文旅、我的 | page, leagueId, source, userId | DAU、路径分析、页面 PV |
| `content_click` | 点击资讯、视频、攻略 | contentId, type, position, source | 内容 CTR、推荐效果 |
| `match_click` | 点击比赛卡、赛程列表 | matchId, leagueId, status | 赛事转化、热门比赛 |
| `booking_click` | 点击预约入口 | matchId, venue, provider | 预约漏斗 |
| `comment_submit` | 发表评论/回复 | targetType, targetId, userId | 社区活跃 |
| `favorite_toggle` | 收藏/取消收藏内容 | targetType, targetId, userId | 我的沉淀、内容收藏率 |
| `follow_toggle` | 关注球队/联赛/比赛 | targetType, targetId, userId | 关注转化 |
| `share_click` | 点击分享 | targetType, targetId, channel | 分享率 |

## 5. 数据看板指标

| 指标 | 口径 | 看板位置 |
|---|---|---|
| DAU | 当日触发任意有效事件的去重用户数 | 总览、数据看板 |
| 平均观看时长 | 页面停留时长总和 / page_view 用户数 | 总览、页面数据 |
| 资讯 CTR | 内容点击数 / 内容曝光数 | 总览、内容上新 |
| 赛事详情转化 | 比赛详情点击数 / 赛事页曝光数 | 赛事管理、数据看板 |
| 预约转化 | 预约成功数 / 预约入口点击数 | 总览、文旅攻略 |
| 评论率 | 评论提交数 / 详情页浏览数 | 社区审核 |
| 收藏率 | 收藏数 / 内容详情浏览数 | 内容上新、我的沉淀 |

## 6. 数据与权限生产化要求

- 数据库：建议 PostgreSQL/MySQL 保存业务数据，Redis 保存验证码、session、队列状态和频控。
- 文件与媒体：封面图、村寨图、视频封面进入对象存储，记录来源、版权、上传人和审核状态。
- 权限：系统管理员、内容运营、赛事数据员、社区审核员、文旅运营、球队管理员使用 RBAC 权限模型。
- 审计：比分修改、内容发布、评论删除、账号禁言、预约配置变更必须写入审计日志。
- 审核：资讯、攻略、社区内容使用“机审 + 人审 + 发布流”，支持驳回原因和版本回滚。
- 数据源：公开抓取、官方供稿、合作 API、人工录入均记录 source_type、source_url、confidence、review_status。

## 7. 当前实现状态

- H5 新增 `/admin` 入口，展示桌面运营后台。
- `apps/api` 新增 `AdminService`，包含管理总览、内容草稿和埋点接收接口。
- Vercel 新增 root `api/admin/overview.ts` 与 `api/analytics/events.ts`，让线上测试版具备基础后端响应。
- 后台首版仍使用 mock/静态数据和 localStorage 草稿，适合产品评审，不等同于生产 CMS。
